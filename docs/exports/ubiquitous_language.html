1. Основные сущности (Core Entities)

1.1. Нейминг (Naming)

Определение: Название, которое предлагается для регистрации в качестве товарного знака или наименования юридического лица (ООО и т.п.).

Контекст: Основной объект анализа в системе. Может быть словесным, комбинированным (слово + изображение) или изобразительным.

Синонимы: Обозначение, название, наименование.

1.2. Товарный знак (Trademark)

Определение: Зарегистрированное обозначение, служащее для индивидуализации товаров и услуг.

Контекст: Элементы из базы данных, с которыми сравнивается проверяемый нейминг.

Синонимы: ТЗ, знак, зарегистрированное обозначение.

1.3. Заявленный нейминг (Submitted Naming)

Определение: Нейминг, который пользователь (юрист) хочет проверить на возможность регистрации.

Контекст: Входные данные для системы.

Синонимы: Проверяемый нейминг, входной нейминг, заявка.

1.4. База данных неймингов (Naming Database)

Определение: Репозиторий зарегистрированных и заявленных товарных знаков, наименований ООО и других обозначений, с которыми может возникнуть конфликт.

Контекст: Источник данных для поиска похожих неймингов.

Синонимы: БД, реестр, база товарных знаков.

1.5. Класс МКТУ (Nice Classification Class)

Определение: Класс товаров и услуг по Ниццкой классификации (Международная классификация товаров и услуг для регистрации знаков).

Контекст: Важный параметр при проверке, так как конфликты обычно рассматриваются в рамках одного или близких классов.

Синонимы: Класс товаров и услуг, МКТУ.

2. Элементы обозначений (Naming Elements)

2.1. Охраняемый элемент (Protectable Element)

Определение: Часть обозначения, которая обладает различительной способностью и может быть защищена законом.

Контекст: Основной фокус при сравнении неймингов.

Синонимы: Доминирующий элемент, значимый элемент.

2.2. Неохраняемый элемент (Non-protectable Element)

Определение: Часть обозначения, которая не обладает различительной способностью и не может быть защищена (цифры, простые указания товаров, общепринятые аббревиатуры и т.п.).

Контекст: Такие элементы игнорируются или учитываются с минимальным весом при сравнении.

Синонимы: Описательный элемент, незначимый элемент.

2.3. Словесный элемент (Verbal Element)

Определение: Текстовая часть обозначения (слово или словосочетание).

Контекст: Основной объект анализа для словесных и комбинированных знаков.

2.4. Изобразительный элемент (Visual Element)

Определение: Графическая часть обозначения (логотип, изображение, цветовое решение).

Контекст: Учитывается при анализе комбинированных знаков.

2.5. Фонетическое ядро (Phonetic Core)

Определение: Ключевая часть обозначения, определяющая его звучание (например, первые слоги или основа слова).

Контекст: Особенно важно при фонетическом сравнении (пример: «PROBI» в «PROBI», «ПРОБИ-БАКТ», «ПРОБИМАКС»).

3. Типы сходства (Similarity Types)

3.1. Тождество (Identity)

Определение: Полное совпадение всех элементов двух обозначений.

Контекст: Максимальный уровень конфликта. Тождественные обозначения не могут быть зарегистрированы для одних и тех же классов МКТУ.

3.2. Сходство до степени смешения (Confusing Similarity)

Определение: Обозначения ассоциируются друг с другом в целом, несмотря на отдельные отличия. Оценивается по общему впечатлению среднего потребителя.

Контекст: Основной критерий для блокировки регистрации.

Синонимы: Сходство, похожесть.

3.3. Фонетическое сходство (Phonetic Similarity)

Определение: Сходство по звучанию обозначений (наличие близких/совпадающих звуков, слогов, их расположение, состав гласных/согласных, ударение и т.п.).

Контекст: Один из трех основных типов сходства.

Примеры: EUROPLEX – EUROFLEX, TIGOTA – DICOTA.

3.4. Графическое сходство (Graphic Similarity)

Определение: Сходство по визуальному восприятию (шрифт, тип букв, расположение, алфавит, цвет и т.п.).

Контекст: Один из трех основных типов сходства.

3.5. Семантическое сходство (Semantic Similarity)

Определение: Сходство по смыслу, значению, понятиям, заложенным в обозначениях (включая значения в разных языках и даже противоположные понятия).

Контекст: Один из трех основных типов сходства.

3.6. Общее впечатление (Overall Impression)

Определение: Интегральная оценка сходства обозначений, формируемая на основе всех типов сходства (фонетического, графического, семантического) и доминирующих элементов.

Контекст: Финальный критерий для определения сходства до степени смешения.

4. Процессы и операции (Processes & Operations)

4.1. Проверка нейминга (Naming Check)

Определение: Процесс анализа заявленного нейминга на предмет возможности его регистрации.

Контекст: Основная бизнес-операция системы.

Синонимы: Анализ нейминга, валидация нейминга.

4.2. Поиск похожих неймингов (Similar Naming Search)

Определение: Операция поиска в базе данных неймингов, схожих с заявленным по фонетике, графике и/или семантике.

Контекст: Ключевая техническая операция системы.

4.3. Сравнение обозначений (Designation Comparison)

Определение: Процесс оценки сходства двух обозначений по всем типам сходства (фонетическое, графическое, семантическое) с учетом охраняемых и неохраняемых элементов.

Контекст: Базовая операция для определения конфликтов.

4.4. Оценка различительной способности (Distinctiveness Assessment)

Определение: Определение того, обладает ли элемент обозначения различительной способностью (может ли он служить для индивидуализации товаров/услуг).

Контекст: Используется для выделения охраняемых и неохраняемых элементов.

4.5. Нормализация обозначения (Designation Normalization)

Определение: Процесс приведения обозначения к стандартному виду для сравнения (удаление неохраняемых элементов, приведение к единому регистру, транслитерация и т.п.).

Контекст: Подготовительный этап перед сравнением.

Определение: Численная оценка степени сходства двух обозначений (обычно от 0 до 1 или в процентах).

Определение: Метрика, оценивающая звуковое сходство обозначений (может использовать фонетические кодирования типа Soundex/Metaphone, анализ слогов, звуков и т.п.).

Контекст: Один из компонентов общей метрики сходства.

5.4. Семантическая метрика (Semantic Metric)

Определение: Метрика, оценивающая смысловую близость обозначений (может использовать эмбеддинги слов, синонимы, переводы и т.п.).

Контекст: Один из компонентов общей метрики сходства.

5.5. Интегральная метрика (Composite Metric)

Определение: Комбинированная метрика, объединяющая фонетические, графические, семантические и лингвистические оценки в единую оценку общего впечатления.

Контекст: Финальная метрика для ранжирования результатов.

5.6. Релевантность (Relevance)

Определение: Степень соответствия найденного нейминга из базы данных запросу пользователя с точки зрения возможности конфликта при регистрации.

Контекст: Используется для ранжирования результатов поиска.

6. Результаты и конфликты (Results & Conflicts)

6.1. Конфликтующий нейминг (Conflicting Naming)

Определение: Нейминг из базы данных, который может помешать регистрации заявленного нейминга из-за тождества или сходства до степени смешения.

Контекст: Основной тип результатов, возвращаемых системой.

Синонимы: Мешающий нейминг, блокирующий нейминг, конфликт.

6.2. Результат поиска (Search Result)

Определение: Элемент из базы данных, найденный в результате поиска похожих неймингов, с указанием метрики сходства и типа конфликта.

Контекст: Единица результата, возвращаемого системой.

6.3. Список конфликтов (Conflict List)

Определение: Упорядоченный список из ~200 наиболее релевантных конфликтующих неймингов для заявленного обозначения.

Контекст: Основной выходной результат системы.

6.4. Риск конфликта (Conflict Risk)

Определение: Оценка вероятности того, что заявленный нейминг будет заблокирован из-за наличия похожих обозначений в базе.

Контекст: Может быть дополнительной информацией для юристов.

7. Специальные термины из юридической практики

7.1. Различительная способность (Distinctiveness)

Определение: Способность обозначения служить для индивидуализации товаров/услуг конкретного производителя.

7.2. Родовое наименование (Generic Term)

Определение: Простое указание товара или услуги, не обладающее различительной способностью (например, «фрукты», «одежда», «ремонт»).

Контекст: Типичный пример неохраняемого элемента.

7.3. Общепринятая аббревиатура (Common Abbreviation)

Определение: Широко используемое сокращение (ООО, ОАО, Ltd, GmbH, НПО, КБ и т.п.), не обладающее различительной способностью.

Контекст: Типичный пример неохраняемого элемента.

7.4. Средний потребитель (Average Consumer)

Определение: Гипотетический потребитель товаров/услуг соответствующей категории, на восприятие которого ориентируется оценка сходства до степени смешения.

Контекст: Референсная точка для субъективной оценки сходства.

7.5. Логическое ударение (Logical Stress)

Определение: Элемент обозначения, на который падает смысловой акцент и который имеет самостоятельное значение.

Контекст: Важен при оценке семантического сходства.
