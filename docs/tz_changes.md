## 4. Уточнение алгоритма работы

### Новое
1. Из базы данных выбираются наименования с совпадающими классами МКТУ.
2. Для каждого кандидата считаются:
   - семантическое сходство;
   - фонетическое сходство;
   - текстовые метрики.
3. На выходе формируется список из `200` наиболее похожих наименований с процентом сходства.
4. Для сценария неправомерного использования:
   - текст сравнивается с одним текстовым наименованием;
   - логотип сравнивается с возможным нарушением;
   - на выходе выдается процент сходства.

**Вывод**: pipeline стал гораздо более формальным и ближе к реальному сервисному поведению.
